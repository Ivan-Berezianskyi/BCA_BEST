# Робота обрахунків метрик і координат
### Отримання даних
 - Обрахунок часу між подіями (gps або imu) в секундах, для того щоб результати отримувати в стандартних одиницях, наприклад m/s
 - Перетворення градусів широти і довготи на радіни, для зручності обчислень
 - обрахунок частоти семплювання за формулою 
 $$\frac{N}{\Delta t}$$
### Фільтрація шумів gps
Для кожного елементу відокремлюємо вікно в 5 секунд або 5 елемнтів, в його межах обраховуємо медіану, звіряємо кожен елемент з медіаною, якщо якийсь елемент відрізняється більше ніж 0.005 градусів ( приблизно 500m в Українських широтах ) значить відбувся збій gps, для чистоти майбутніх обчислень точку потрібно видалити
### Перетворення координат
Прямого переведення WGS-84 в ENU немає, тому необхідно скористатись проміжним етапом в вигляді ECEF. Алгоритм працює таким чином, спочатку переводимо WGS-84 в метри від центру маси землі. Після чого шукаємо різницю кожної точки відносно опорної ( В нашому випадку точка старту ), останій етап поворот системи координат щоб відновити напрямок вгору
### Фільтрація координат на основі даних в ENU
Додатковий етап фільрації координат, який заміряє швидкість переміщення дрона між двома точками 
$$\frac{\sqrt{(x_1-x_2)^2 + (y_1-y_2)^2 + (z_1-z_2)^2}}{\Delta t}$$

### Обрахунок дистанції
Використання Haversine дозволяє моделювати реальне переміщення повторюючи геометрію планети, забезпечуючи кращу точність на дистанція > 20 km ніж різниця координат ENU
$$a = \sin^2\left(\frac{\Delta\phi}{2}\right) + \cos(\phi_1)\cos(\phi_2)\sin^2\left(\frac{\Delta\lambda}{2}\right)$$
$$c = 2 \operatorname{atan2}\left(\sqrt{a}, \sqrt{1-a}\right)$$
$$d = R \cdot c$$

### Обрахунок кватеріонів повороту
Для точності обрахунків швидкості, необхідні підготовлені дані акселерометра, а саме необхідно відняти прискорення вільного падіння, але так як вектор прискорення вираховується не від землі, а від положення корпусу дрона його необхідно вирівняти по горизонту. Саме для цього потрібні кватеріони, які на відміно від поступового повороту осей, запобігають злиттю двох осей ( Gimbal lock ). Так як тз дозволяє користуватись лише записами GPS і IMU, для їх обрахунку були використанні записи кутової швидкості. Для отримання кута повороту потрібно її інтегрувати але це накопичує похибку. Щоб запобігти цьому використано фільтр Маджвіка який коригує дані завдяки постійному прискореню вільного падіння на акселерометрі.

### Обрахунок прискорення
Повертаємо вектор прискорення, застосовуєм функцію detrend яка віднімає похибку накладену не точностями акселерометра ( критично для обрахунку швидкості через інтегрування )

### Обрахунок швидкості
Рахуємо швидкість методом трапецієвидного інтегрування

### Синхронізація часу
Так як частота семплінгу в gps та imu різна, для їх злиття необхідно, або розтягнути дані gps на дані imu або обрізати дані imu до даних gps. Був вибраний другий варіант по причині того що відображення змін за наприклад 0.5 секунд в прискорені не є критичними, і майбутнє форматування графіку базується саме за даними gps
